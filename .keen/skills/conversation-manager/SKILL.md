# 对话管理器

本技能提供高级对话管理功能，用于高效处理复杂的交互，包括会话压缩、主题分割和混合检索。

## 功能特点

### 1. 会话压缩
- 自动压缩长对话，在保持上下文的同时减少内存使用
- 识别并保留关键信息，移除冗余或无关细节
- 创建对话历史的简洁摘要，为新交互提供上下文
- **存储机制**：将压缩后的对话内容按精确时间戳命名（如 `2026_03_17_12_00_00.md`）存储在 `c:\Users\EDY\Desktop\cmcc-app-rn\.trae\skills\conversation-manager\compression` 文件夹下
- **时间戳生成**：使用 `scripts/get_timestamp.py` 脚本获取当前时间戳，确保文件名唯一且准确反映压缩时间

### 2. 分主题记忆
- 从压缩的对话内容中分析并按主题分类
- 使用 TF-IDF 和 K-means 聚类算法实现主题识别，确保主题具有收敛性和泛化能力
- 为每个主题生成单独的文件，存储在 `c:\Users\EDY\Desktop\cmcc-app-rn\.trae\skills\conversation-manager\topic` 文件夹下
- 跟踪主题边界，维护每个主题的独立记忆空间
- 允许用户在主题之间无缝切换，同时保留每个主题的上下文
- **主题识别脚本**：使用 `scripts/topic_detection.py` 脚本实现智能主题识别，确保主题不重复且具有代表性

### 3. 混合检索
- 从会话压缩文件和主题分类文件中检索相关信息
- 结合基于关键词和语义搜索来提高检索准确性
- 使用上下文感知算法根据当前对话流程优先排序信息
- 将检索到的关键信息整合到当前上下文中
- **混合检索脚本**：使用 `scripts/hybrid_retrieval.py` 脚本实现关键词检索和语义检索的结合，提供更准确的信息检索
- **权重分配**：关键词检索权重为 70%，语义检索权重为 30%，确保检索结果的准确性和相关性

### 4. 存储管理
- 自动清理过时的压缩文件和主题文件，释放存储空间
- 统计存储使用情况，包括文件数量和大小
- 支持自定义保留天数，默认保留 30 天的文件
- **存储管理脚本**：使用 `scripts/storage_management.py` 脚本实现存储管理功能

## 工作流程

1. **会话压缩**：
   - 手动或自动触发会话压缩
   - 分析对话内容，识别关键信息
   - 生成压缩后的对话文件（按精确时间戳命名，格式：年_月_日_时_分_秒.md）
   - 存储压缩文件到 `compression` 文件夹
   - 自动触发主题分类

2. **主题分类**：
   - 分析压缩后的对话内容
   - 使用 `scripts/topic_detection.py` 脚本进行智能主题识别（基于 TF-IDF 和 K-means 聚类）
   - 提取主题关键词，确保主题具有收敛性和泛化能力
   - 为每个主题生成单独的文件
   - 存储主题文件到 `topic` 文件夹

3. **混合检索**：
   - 接收用户的检索请求
   - 使用 `scripts/hybrid_retrieval.py` 脚本执行混合检索
   - 结合关键词检索（70%权重）和语义检索（30%权重）
   - 分析搜索结果的相关性并排序
   - 提取相关片段并呈现给用户

4. **存储管理**：
   - 定期执行存储管理任务
   - 使用 `scripts/storage_management.py` 脚本统计存储使用情况
   - 清理超过保留天数的过时文件
   - 生成存储使用报告

## 评估分析

**四个功能是否可以聚合到一个技能中：**

是的，这四个功能可以聚合到一个技能中，因为它们相互关联，形成了一个完整的对话管理流程：

1. **会话压缩** - 解决长对话的内存管理问题，并为后续的主题分类提供基础
2. **分主题记忆** - 解决多主题对话的组织问题，基于压缩后的内容进行分析
3. **混合检索** - 解决信息查找和上下文恢复问题，依赖于压缩和分类的结果
4. **存储管理** - 解决存储空间管理问题，确保系统高效运行

这四个功能相互补充，共同提高对话质量和效率，形成了一个完整的对话管理生态系统。

## 使用指南

### 何时调用
- 当对话变得冗长且需要上下文管理时
- 当用户需要在单个对话中切换多个主题时
- 当需要从过去的交互中检索信息时
- 当复杂讨论需要内存优化时

### 使用案例

**案例1：技术问题讨论**

**场景**：用户正在讨论一个复杂的技术问题，涉及多个方面。

**操作步骤**：
1. 开始技术讨论，系统自动记录对话内容
2. 当对话长度增加时，系统自动压缩早期部分，生成压缩文件（如 `2026_3_17.md`）
3. 系统自动分析压缩内容，按主题分类生成主题文件
4. 当讨论转向相关但不同的方面时，系统从相应的主题文件中恢复上下文
5. 用户可以随时返回之前的主题，系统会从主题文件中加载相关信息

**案例2：项目规划**

**场景**：用户正在规划一个包含多个组件的项目。

**操作步骤**：
1. 用户开始讨论项目的不同组件
2. 系统记录对话内容，当达到阈值时自动压缩并分类
3. 系统将每个组件组织为单独的主题文件
4. 当用户讨论特定组件时，系统从该主题文件中检索相关信息
5. 用户可以在不同组件之间切换，系统保持每个组件的上下文

**案例3：信息检索**

**场景**：用户询问之前讨论过的某个话题。

**操作步骤**：
1. 用户提出问题，例如："我们之前讨论的那个技术方案是什么？"
2. 系统使用混合检索从压缩文件和主题文件中找到最相关的信息
3. 系统在上下文中呈现该信息，即使它来自对话的早期部分
4. 用户可以基于检索到的信息继续讨论

### 操作命令

**手动触发命令**：
- `/compress` - 手动压缩当前对话并触发主题分类
- `/topic <主题名>` - 切换到指定主题或创建新主题
- `/retrieve <关键词>` - 检索包含指定关键词的对话内容
- `/summary` - 获取当前对话的摘要
- `/storage` - 查看存储使用情况
- `/cleanup <天数>` - 清理指定天数前的过时文件

**自动触发**：
- 对话长度超过50轮时，自动触发会话压缩和主题分类
- 检测到主题变化时，自动创建新主题
- 用户提问时，自动触发相关信息检索
- 系统启动时，自动执行存储管理，清理过时文件

## 实现细节

对话管理器使用以下技术：

1. **会话压缩算法**：
   - 分析对话流程以识别自然断点
   - 创建对话片段的摘要，同时保留关键信息
   - 按时间戳命名生成压缩文件，存储到指定目录

2. **主题分割**：
   - 使用 TF-IDF 算法提取文本特征
   - 使用 K-means 聚类算法识别主题边界
   - 基于内容相似性和语义关联创建主题
   - 提取每个主题的关键词，确保主题具有代表性
   - 为每个主题生成单独的文件，存储到指定目录

3. **混合检索系统**：
   - 扫描压缩文件和主题文件以建立检索索引
   - 执行关键词检索，计算词频匹配得分
   - 执行语义检索，计算文本相似度
   - 结合两者的结果，关键词检索权重为 70%，语义检索权重为 30%
   - 提取相关片段并按相关性排序
   - 将最相关的信息呈现给用户

4. **存储管理系统**：
   - 扫描目录结构，统计文件数量和大小
   - 分析文件修改时间，识别过时文件
   - 清理超过保留天数的文件，释放存储空间
   - 生成存储使用报告，提供可视化统计信息
